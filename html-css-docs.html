Anchor — Âncora / Hiperlink

Define um hiperlink, que conecta uma página a outra ou a uma seção dentro da mesma página. É o elemento fundamental da web, permitindo navegação entre documentos, downloads de arquivos, links para e-mails (mailto:), chamadas telefônicas (tel:) e âncoras internas via fragmentos (#id). Quando não possui o atributo href, comporta-se como um marcador de posição (placeholder).

Atributos
★ Aceita todos os atributos globais (id, class, style, title, lang, data-*, etc.)
AtributoValores possíveisDescrição
hrefURL | #id | mailto: | tel: | javascript:Define o destino do link. Obrigatório para links funcionais. Pode ser absoluto, relativo, âncora, esquema de protocolo ou URI de dados.
target_self | _blank | _parent | _top | <nome>_self: abre no mesmo contexto (padrão). _blank: nova aba/janela. _parent: frame pai. _top: janela de nível mais alto. Nome personalizado abre em frame nomeado.
relnoopener | noreferrer | nofollow | sponsored | ugc | me | alternate | author | bookmark | help | license | next | prev | search | tagDefine a relação entre o documento atual e o destino. noopener: segurança com _blank. noreferrer: não envia Referer. nofollow: SEO — não siga este link.
downloadfilename | (vazio)Indica que o destino deve ser baixado em vez de aberto. O valor opcional define o nome do arquivo sugerido para download.
hreflangcódigo BCP 47 (ex: pt-BR, en-US)Especifica o idioma do documento de destino. Útil para SEO e acessibilidade em sites multilíngues.
typeMIME type (ex: text/html, application/pdf)Indica o tipo de mídia do recurso de destino. Apenas informativo para o navegador.
pinglista de URLs separadas por espaçoQuando o link é clicado, o navegador envia requisições POST para as URLs listadas. Usado para rastreamento de cliques.
referrerpolicyno-referrer | no-referrer-when-downgrade | origin | origin-when-cross-origin | same-origin | strict-origin | strict-origin-when-cross-origin | unsafe-urlControla quanta informação do Referer é enviada ao seguir o link.

referrerpolicyno-referrer | origin | etc.Controla as informações do Referer enviadas.

Audio — Elemento de Áudio Embutido

Incorpora conteúdo de áudio diretamente na página sem necessidade de plugin externo. Suporta múltiplos formatos via elementos <source> filhos (MP3, OGG, WAV, AAC, FLAC). O navegador escolhe o primeiro formato suportado. Pode conter conteúdo de fallback para navegadores sem suporte. Oferece controles nativos do navegador ou pode ser controlado via API JavaScript (Web Audio API).

Atributos
★ Aceita todos os atributos globais
AtributoValores possíveisDescrição
srcURLURL do arquivo de áudio. Alternativa ao uso de <source> interno. Se usar <source>, omitir este atributo.
controls(booleano)Exibe os controles nativos do navegador (play/pause, volume, progresso, download). Sem este atributo, o áudio é invisível e deve ser controlado via JS.
autoplay(booleano)Inicia a reprodução automaticamente. Muitos navegadores bloqueiam autoplay sem interação do usuário (especialmente com som). Use com muted para garantir funcionamento.
loop(booleano)Reproduz o áudio em loop contínuo ao terminar.
muted(booleano)Inicia o áudio no estado mudo. O usuário pode desmutar manualmente. Permite autoplay na maioria dos navegadores.
preloadnone | metadata | autonone: não pré-carrega. metadata: carrega apenas duração/metadados. auto (padrão): carrega o arquivo inteiro. Afeta a performance da página.
crossoriginanonymous | use-credentialsHabilita CORS para o recurso. anonymous: sem credenciais. use-credentials: envia cookies/certificados.
Exemplos de Uso
HTML
<!-- Áudio simples com controles --> <audio controls> <source src="musica.mp3" type="audio/mpeg"> <source src="musica.ogg" type="audio/ogg"> Seu navegador não suporta o elemento audio. </audio> <!-- Áudio em loop sem exibir controles --> <audio src="ambient.mp3" autoplay muted loop preload="auto"> </audio>

Bold — Texto em Negrito (sem ênfase semântica)

Representa texto que deve ser chamado à atenção do leitor sem transmitir importância semântica adicional ou ênfase alterada. Tipicamente renderizado em negrito. Diferente de <strong>, que indica importância; e de <em>, que indica ênfase de pronunciação. Use <b> para: palavras-chave em um resumo, nomes de produtos em uma review, texto acionável. Para importância real, use <strong>.

Atributos
★ Aceita apenas atributos globais

Não possui atributos específicos.

Exemplos de Uso
HTML
<p>O produto <b>ProMax 5000</b> tem excelente custo-benefício.</p> <!-- Vs ênfase semântica com strong --> <p><strong>Atenção:</strong> este processo é irreversível.</p>

Base — URL Base e Target Padrão do Documento

Define a URL base para todas as URLs relativas no documento e/ou o target padrão para todos os hiperlinks. Deve ser o primeiro elemento no <head> com uma URL, pois afeta todos os elementos após ele. Só pode haver um único <base> por documento. Afeta: <a>, <img>, <link>, <script>, <form>, <video>, <audio> e outros elementos com URLs.

Atributos
★ Aceita atributos globais
AtributoValores possíveisDescrição
hrefURL absolutaDefine a URL base para todas as URLs relativas no documento. Deve ser uma URL absoluta. Ex: https://example.com/pasta/
target_self | _blank | _parent | _top | <nome>Define o contexto de navegação padrão para todos os links que não especificam seu próprio target.
Exemplos de Uso
HTML
<head> <base href="https://example.com/docs/" target="_blank"> </head> <body> <!-- Este link resolve para: https://example.com/docs/intro.html --> <a href="intro.html">Introdução</a> <!-- Esta imagem resolve para: https://example.com/docs/logo.png --> <img src="logo.png" alt="Logo"> </body>

Bidirectional Isolate — Isolamento Bidirecional

Isola um trecho de texto que pode ter uma direcionalidade diferente do texto ao redor. Essencial para páginas multilíngues que misturam texto RTL (árabe, hebraico, persa) com texto LTR (latim, etc.). Impede que a direcionalidade do texto interno afete o texto circundante. Diferente de <bdo> que força a direção; <bdi> isola e detecta automaticamente.

Atributos
★ Aceita atributos globais. O atributo dir é especialmente relevante.
AtributoValores possíveisDescrição
dirltr | rtl | autoDefine a direção do texto. ltr: esquerda para direita. rtl: direita para esquerda. auto (padrão para bdi): detecta automaticamente pela heurística do primeiro caractere forte.
Exemplos de Uso
HTML
<!-- Nome de usuário pode conter caracteres RTL --> <ul> <li>Usuário <bdi>إيان</bdi>: 90 pontos</li> <li>Usuário <bdi>Carlos</bdi>: 75 pontos</li> </ul>

Bidirectional Override — Substituição Bidirecional

Substitui (força) a direcionalidade do texto, ignorando o algoritmo bidirecional Unicode e exibindo os caracteres na direção especificada. Usado quando a direção do texto é conhecida e deve ser aplicada explicitamente, independente dos caracteres. Ao contrário de <bdi>, que isola e detecta; <bdo> impõe a direção.

Atributos
AtributoValores possíveisDescrição
dirltr | rtlObrigatório. ltr: força esquerda para direita. rtl: força direita para esquerda (espelha o texto visualmente).
Exemplos de Uso
HTML
<p>Texto normal: Olá Mundo!</p> <p>Espelhado: <bdo dir="rtl">Olá Mundo!</bdo> </p>

Line Break — Quebra de Linha

Produz uma quebra de linha dentro de um texto, sem iniciar um novo parágrafo. Útil onde quebras de linha são significativas: poesia, endereços postais, letras de música. Não use <br> para criar espaçamento entre parágrafos — use CSS (margin). É um elemento vazio (void) sem tag de fechamento. Aceita apenas atributos globais.

Atributos
★ Aceita apenas atributos globais

Não possui atributos específicos. O atributo clear (obsoleto) era usado para controlar o comportamento com floats — use CSS hoje.

Button — Botão Interativo Clicável

Representa um botão clicável interativo. Muito mais semântico e acessível que <div> ou <span> clicáveis. Pode conter conteúdo HTML (texto, ícones, imagens). Dentro de formulários, envia (type="submit"), reseta (type="reset") ou executa JS (type="button"). Suporta estados disabled, focus, e é acessível por teclado nativamente. Preferido ao <input type="submit"> por aceitar HTML interno.

Atributos
★ Aceita todos os atributos globais + atributos de evento
AtributoValores possíveisDescrição
typesubmit | reset | buttonsubmit: envia o formulário pai (padrão dentro de <form>). reset: redefine os campos do formulário. button: sem ação padrão — controlado via JS. Sempre especifique para evitar comportamento inesperado.
disabled(booleano)Desativa o botão, impedindo interação e envio. Aparência acinzentada. Não recebe foco por teclado.
nametextoNome do botão submetido com o formulário. Permite identificar qual botão foi clicado quando há múltiplos.
valuetextoValor enviado ao servidor quando o botão com name é submetido.
formid do <form>Associa o botão a um formulário específico pelo ID, mesmo estando fora dele no HTML.
formactionURLSobrescreve o atributo action do formulário para este botão específico.
formmethodget | post | dialogSobrescreve o método HTTP do formulário para este botão.
formenctypeapplication/x-www-form-urlencoded | multipart/form-data | text/plainSobrescreve o tipo de codificação do formulário para este botão.
formnovalidate(booleano)Ignora a validação HTML5 do formulário ao submeter com este botão.
formtarget_self | _blank | _parent | _topSobrescreve o target do formulário para este botão.
autofocus(booleano)Dá foco automático ao botão quando a página carrega. Deve ser usado com cautela para acessibilidade.
popovertargetid do popover(HTML 6.1+) Associa o botão como controlador de um elemento popover.
popovertargetactiontoggle | show | hideDefine a ação do botão no popover associado.
Exemplos de Uso

Canvas — Área de Desenho Programático

Define uma área de desenho bitmap controlada via JavaScript. Todo o conteúdo é desenhado com a API canvas.getContext('2d') ou getContext('webgl'). Usado para gráficos, animações, jogos e processamento de imagem. O conteúdo interno é fallback para navegadores sem suporte.

Atributos
★ Aceita todos os atributos globais
AtributoValoresDescrição
widthinteiro (px)Largura do bitmap interno. Padrão: 300. Diferente da largura CSS — define a resolução do canvas, não o tamanho de exibição.
heightinteiro (px)Altura do bitmap interno. Padrão: 150. Mesma distinção acima.
